1. Explain what state management is and why it’s important in Flutter applications.

ans->State management is the process of managing and updating the data (state) of a Flutter application. 
A state is any information that can change while the app is running, such as a counter value, user login status, 
or shopping cart items.

->Importance of State Management:
  Keeps the UI updated when data changes.
  Makes the code clean and organized.
  Allows data to be shared between multiple widgets.
  Improves app performance.
  Makes large applications easier to maintain.

3. Describe the Provider package and how it differs from basic setState usage.
Provider is a Flutter package used for state management. It allows multiple widgets to access and update shared data efficiently without passing data manually between widgets.

->>Advantages of Provider
   Easy to use.
   Keeps code organized.
   Shares data across multiple widgets.
   Automatically rebuilds widgets when data changes.
   Suitable for medium and large applications.
   Difference between Provider and setState()

->      setState()	                     Provider

     Built into Flutter	                 External package
     Best for small apps	         Best for medium and large apps
     Updates only one widget	         Updates multiple widgets using shared data
     Difficult to share data	         Easy to share data across the app
     Can make code messy in large apps	 Keeps code clean and maintainable
